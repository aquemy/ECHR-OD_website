ECHR OpenData (ECHR-OD) project

The project

The European Court of Human Rights OpenData project aims at providing formatted and standardized open data about the European Court of Human Rights in order to draw the attention of Data Scientists, Machine Learning practionners and in general researchers on legal analysis and associated problems.

Currently, the project focuses on two objectives:

  • Analyzing and extracting standardized information for each cases available in the HUDOC database to provide the most comprehensible open database on the ECHR.
  • Providing several datasets for the Machine Learning Classification problems.

    • Contributing

    ECHR-OD accepts contributions under various forms:
  • Visualization and data exploration
  • Reports of misformed or incorrectly processed data (see here)
  • Datasets for any type of problems related to Machine Learning, Knowledge Management, Data Science,...

    Descriptive variables

    The descriptive variables are describing the cases. They are provided by HUDOC directly for most, or extracted from the judgement document for some.

    Name ID Description

    Outcomes variables

    The outcomes variables are binary variables indicating if there is a violation (value 1) or no violation (value 0) of an article. If a variable is absent from a case, it means the case was not about this article. The variable with ID 0 correspond to the case ID for reference purpose.

    Article ID Description
    Example: 
    0:8707 12:1 3:0 4:0
    This indicated that the the judgement of case ID 8707 states a violation of the article with ID 12 and no violation for the articles with ID 3 and 4.

    Datasets

    For each dataset, several versions are available:
  • Descriptive features only
  • Bag-of-Word only
  • TF-IDF only
  • Descriptive features + Bag-of-Words
  • Descriptive features + TF-IDF

    • Multi-label Classification Dataset

    The multi-label classification consists in finding a map that assign to each input vector one or several labels. The dataset contains instances and output variables, each of them taking value into {0,1}.

    Warning: each output variable indicated that a specific article is violated (1) or not (0). If a variable does not appear in the list for a specific case it means the article was not discussed.
